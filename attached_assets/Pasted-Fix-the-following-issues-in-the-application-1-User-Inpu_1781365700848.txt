Fix the following issues in the application.

## 1. User Input Validation

Do not allow users to register with invalid information.

### Full Name

* Required.
* Minimum 3 characters.
* Only alphabets and spaces are allowed.

Show error:
"Please enter a valid name."

---

### Email Validation

Validate email format.

Examples of valid emails:
[user@gmail.com](mailto:user@gmail.com)
[abc@yahoo.com](mailto:abc@yahoo.com)

Examples of invalid emails:
abc
gmail.com
abc@

If invalid, show:
"Please enter a valid email address."

Registration should not proceed until a valid email is entered.

---

### Phone Number Validation

Allow only Indian mobile numbers.

Requirements:

* Exactly 10 digits.
* Must start with 6, 7, 8, or 9.
* No letters or symbols.

Examples:

Valid:
9876543210

Invalid:
1234567890
abcd12345
99999

Show error:
"Please enter a valid Indian mobile number."

---

### OTP Verification

Access should be granted only after successful OTP verification.

Wrong OTP:
Show:
"Invalid OTP"

Expired OTP:
Show:
"OTP expired"

Maximum attempts:
3

After 3 failed attempts:
Lock verification for 5 minutes.

---
